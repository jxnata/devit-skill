Development without structure is expensive.

When Claude has no plan, tasks drift in scope, verification gets skipped, and the same problems resurface. Structure changes that.

Context drift

Without a defined scope, implementations grow beyond the task boundary and break unrelated functionality in ways that are hard to trace.

Skipped verification

Tests, lint, and type checks get bypassed under pressure. Bugs reach production that a single verification pass would have caught.

Incomplete features

Large features without a task breakdown often stall. No clear definition of done means no clear finish line and no accountability.

How it works

Three commands. A complete workflow.

devit structures every feature into a repeatable, verifiable cycle. From zero to shipped.

01
/devit start

Initialize your project

devit reads your manifest files and lockfiles to detect your stack, package manager, test runner, linter, and type checker. It writes project-adapted workflow guides into .devit/.

+ Stack auto-detection
+ Command resolution
+ .devit/ scaffold
02
/devit plan [feature]

Break it into tasks

Describe a feature in natural language. devit creates an epic with ordered, atomic 1-2 hour tasks. Each task includes TDD steps, expected test cases, acceptance criteria, and the exact files to touch.

+ Epic + task JSON
+ TDD steps per task
+ Dependency ordering
03
/devit execute

Execute with verification

devit runs a 6-step per-task workflow: write failing tests, implement, verify (type-check + lint + tests pass), commit with a structured message, and update task status. Done is never declared without passing verification.

+ 6-step workflow
+ Mandatory verification
+ Auto commit + status
Model recommendation
Use Opus for /devit plan to get the best task breakdown. Switch to Sonnet for /devit execute for fast, focused implementation.
Features

Built for real development workflows.

Every design decision in devit comes from the friction of building features without structure.

TDD enforcement

Tests before code

Every task begins with a failing test. The execution flow refuses to move forward without one. No exceptions, no shortcuts.

Stack detection

Works with your toolchain

Reads manifest files and lockfiles to find your package manager, test runner, linter, and type checker. Works with Node, Python, Go, and Rust.

Verification gates

Definition of done, enforced

Type-check, lint, and all tests must pass before a task is marked complete. Partial success is not success.

Atomic tasks

1-2 hour scoping

Tasks are sized to be completable in a single focused session. No ambiguous chunks that grow into multi-day rabbit holes.

Simple task flow

No overhead for small fixes

Quick bug fixes and isolated changes skip epic scaffolding entirely. Same TDD discipline, zero boilerplate.

Commit format

Traceable history

Every commit is tagged with its epic and task: feat: description (E03 task 2). Status and commit hash tracked in meta.json.

Task format

Every task tells Claude exactly what to build.

Each task JSON includes TDD steps, concrete test cases, acceptance criteria, and the exact files to touch. Claude has everything it needs without a long prompt chain.

Steps array
TDD-ordered steps: write failing test, create file, implement, refactor.
Tests array
Specific test cases: happy path, validation errors, edge cases.
Acceptance criteria
Explicit conditions that must be true before the task is considered done.
Files list
Exactly which files to modify and which to create. Scope stays contained.
.devit/epics/3/tasks/1.json 
{
  "id": 1,
  "title": "Setup OAuth provider config",
  "description": "Create config module with
  provider-specific settings.",
  "steps": [
    "Write failing test for getAuthConfig()",
    "Create src/auth/config.ts",
    "Define AuthConfig interface",
    "Implement to pass tests",
    "Refactor safely"
  ],
  "tests": [
    "returns config for valid provider",
    "throws on unknown provider",
    "handles missing env vars"
  ],
  "acceptanceCriteria": [
    "All tests pass",
    "No TypeScript errors",
    "No lint warnings"
  ],
  "files": {
    "create": [
      "src/auth/config.ts",
      "src/auth/config.test.ts"
    ]
  }
}
Installation

One command to install. Updates via git pull.

Installed as a symlink, so pulling the latest changes updates the skill instantly. No re-install needed.

One-line install
$ git clone https://github.com/jxnata/devit-skill.git && cd devit-skill && bash install.sh
Then in any project
$ /devit start
Keep it updated
$ cd devit-skill && git pull
