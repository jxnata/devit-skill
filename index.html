Development without structure is expensive.

When Claude has no plan, scope drifts, verification gets skipped, and the same bugs come back.

Context drift

Implementations grow beyond the task boundary and break unrelated functionality.

Skipped verification

Tests and type checks get bypassed under pressure. Bugs reach production.

Incomplete features

No task breakdown, no finish line. Large features stall and stay half-done.

How it works

Three commands. Complete workflow.

A repeatable, verifiable cycle from zero to shipped.

01
/devit start

Initialize your project

Reads manifests and lockfiles to detect your stack and commands. Writes .devit/ guides adapted to your project.

+ Stack auto-detection
+ Command resolution
+ .devit/ scaffold
02
/devit plan [feature]

Break it into tasks

Describe a feature. devit creates an epic with atomic 1-2 hour tasks including TDD steps, test cases, and acceptance criteria.

+ Epic + task JSON
+ TDD steps per task
+ Dependency ordering
03
/devit execute

Execute with verification

6-step workflow: write failing tests, implement, verify (type-check + lint + tests), commit, and update task status. Done is never declared without passing verification.

+ 6-step workflow
+ Mandatory verification
+ Auto commit + status
Model tip

Use Opus for /devit plan and Sonnet for /devit execute.

Features

Built for real development.

Every decision comes from the pain of building features without structure.

TDD enforcement

Tests before code

Every task starts with a failing test. No exceptions, no shortcuts.

Stack detection

Works with your toolchain

Detects your package manager, test runner, linter, and type checker. Node, Python, Go, Rust.

Verification gates

Done means done

Type-check, lint, and all tests must pass. Partial success is not success.

Atomic tasks

1-2 hour scoping

Tasks sized for a single focused session. No ambiguous chunks that never finish.

Simple task flow

No overhead for small fixes

Small changes skip epic scaffolding entirely. Same discipline, zero boilerplate.

Commit format

Traceable history

Every commit tagged: feat: desc (E03 task 2). Status tracked in meta.json.

Task format

Every task tells Claude exactly what to build.

Each task JSON has TDD steps, test cases, acceptance criteria, and the files to touch. Claude needs no extra prompting.

Steps array
TDD-ordered: write failing test, create file, implement, refactor.
Tests array
Happy path, edge cases, validation errors.
Acceptance criteria
Explicit conditions that must all be true before the task is done.
Files list
Exactly which files to modify and create. Scope stays contained.
.devit/epics/3/tasks/1.json 
{
  "id": 1,
  "title": "Setup OAuth provider config",
  "steps": [
    "Write failing test for getAuthConfig()",
    "Create src/auth/config.ts",
    "Define AuthConfig interface",
    "Implement to pass tests",
    "Refactor safely"
  ],
  "tests": [
    "returns config for valid provider",
    "throws on unknown provider",
    "handles missing env vars"
  ],
  "acceptanceCriteria": [
    "All tests pass",
    "No TypeScript errors",
    "No lint warnings"
  ],
  "files": {
    "create": [
      "src/auth/config.ts",
      "src/auth/config.test.ts"
    ]
  }
}
Installation

One command to install. Updates via git pull.

Installed as a symlink. Run git pull to update.

One-line install
$ git clone https://github.com/jxnata/devit-skill.git && cd devit-skill && bash install.sh
Then in any project
$ /devit start
Keep it updated
$ cd devit-skill && git pull
